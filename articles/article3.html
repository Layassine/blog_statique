Git et GitHub

Maîtrisez la gestion de versions pour travailler en équipe efficacement.

Qu'est-ce que Git ?

Git est un système de gestion de versions créé en 2005 par Linus Torvalds (le même créateur de Linux). Il permet de suivre toutes les modifications apportées à un projet, de revenir en arrière en cas d'erreur, et de travailler à plusieurs sur le même code sans se marcher dessus.

Git vs GitHub

Ce sont deux choses différentes mais complémentaires :

  • Git — l'outil installé sur votre ordinateur qui gère les versions
  • GitHub — le site web qui héberge votre dépôt Git en ligne
  • Git fonctionne sans GitHub, mais GitHub a besoin de Git

Les commandes essentielles

Voici les commandes Git que tout développeur doit connaître :

  • git init — initialiser un nouveau dépôt
  • git clone URL — copier un dépôt existant
  • git add fichier — préparer un fichier au commit
  • git commit -m "message" — sauvegarder les modifications
  • git push origin main — envoyer sur GitHub
  • git pull origin main — récupérer les mises à jour
  • git branch nom — créer une nouvelle branche
  • git merge branche — fusionner deux branches

Le workflow en équipe

Dans un projet collaboratif, chaque développeur travaille sur sa propre branche pour éviter les conflits. Une fois son travail terminé, il crée une Pull Request sur GitHub pour demander la fusion de son code dans la branche principale.

  • Chaque membre → sa propre branche
  • Des commits réguliers avec des messages clairs
  • Une Pull Request pour chaque fonctionnalité terminée
  • Un responsable qui valide et merge les PR

Conclusion

Git est aujourd'hui un outil indispensable dans tout projet informatique professionnel. Le maîtriser, c'est s'ouvrir les portes de tous les projets open source et de toutes les entreprises du secteur tech. Commencez par pratiquer sur de petits projets !
